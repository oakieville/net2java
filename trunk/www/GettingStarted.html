Getting Started Using NET2Java
January 4th 2007

Be sure to get the latest downloads: Check for the latest here

install the NET2Java Library
  • make a directory called dnj in your user home directory
  • e.g. on windows create C:\Documents And Settings\Danny\dnj\
  • unzip the NET2Java Library Archive to the directory you just made
install the NET2Java NetBeans Plugin
(I'm really sorry if you don't like NetBeans, I'll put in the command line version of NET2Java soon)
  • In your Netbeans 5.0, from the Tools menu, open the Update Center
  • Choose the option to Install Manually Downloaded Files (.nbm modules), hit Next
  • Add the NET2Java Plugin file, follow the wizard through to the end

the installation is SO over
ready to translate some apps ?

  • unzip these NET samples somewhere handy
  • From NetBeans, create a new project (select the Java Application template for example)
  • Add the path to the .NET2Java library entry files and classes to your new project
  • e.g. In the Project Properties dialog, select the Libraries category and
    and add     C:\Documents And Settings\Danny\dnj\ by hitting the 'Add Jar/Folder' button
  • Import your first .NET project, by choosing File -> ImportVB top bring up the Import Wizard
  • Selecting the <samples>HelloWorldFromVB project directory, select 'Console app' and hit Finish
  • You should see HelloWorldFromVB...in Java


Enjoy the other demos (using this handy guide), have a go with other apps you can find (hint: Microsoft's MSDN has plenty...)

or...did you have problems ?
I'm sorry to hear that :-(
Usually the problem will be that there will be portions of the application
that use .NET API calls that have not yet been entered with a Java equivalent in the Library.
You can check yourself to see what's in the Library by choosing in NetBeans
  • Tools -> .NET Library -> Edit Entries
Maybe you will even work out how to add your own translations to the Library.
